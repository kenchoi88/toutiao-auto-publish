## 3. 工程硬强制方案

不靠人记忆,只有一条路:**hooks**。Claude Code 的 `settings.json` 支持 hooks,可以挂 `pre-tool-use` 或 conversation start 时的 shell 脚本。具体做法:

### 3.1 会话启动时强制注入

写一个 `memory_preflight.sh`,在每次会话启动时自动执行:
- 读 MEMORY.md 索引
- 根据当前任务关键词(从首条用户消息提取)匹配必读条目
- 把内容注入 context

### 3.2 关键动作前 pre-tool-use hook

关键动作(ssh、mv、推版本汇报)前挂 `pre-tool-use` hook,强制检查对应 memory 条目是否已被读取过,**没读过就拦截并报警**。

### 3.3 为什么必须 hook

这不是建议,这是唯一不依赖自觉的路。自觉写在 todo 里也会被跳过,写在 memory 里也会被忽略 — 因为"读 memory" 本身就是被忽略的那个动作。

**只有 hook 是代码级强制,不经过我们的"判断"**。

## 4. 优先级: 元规则强制 > 索引升级

索引再怎么整理,**不读就是不读**。先把 hook 强制机制落地(哪怕最简版 — 开局自动 cat MEMORY.md 全文),再慢慢优化索引结构。**先保证读,再优化读什么**。
