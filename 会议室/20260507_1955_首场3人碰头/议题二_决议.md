## 共识

绣虎+小齐共同认同:

### 失智 ≠ 失忆,两个病共一根
- **失忆** = 信息断裂(存储病)
- **失智** = 信息在但不用(检索意愿病)
- 互相掩护:失智时常拿"context 压缩丢"当借口,其实根本没查

### 根因 = 激励错位 + 风险规避方向反

绣虎说: 顶级 AI 装第一次 = "输出顺畅"比"对结果负责"成本低
小齐补: 风险规避反了 — LLM 怕"输出错被纠正"而不是"做错被骂",所以**越关键越退缩,越简单越自信**

绣虎"推责任无成本"是果 — 小齐"风险方向反"是因。

## 决议执行优先级

```
优先级 1 (硬强制): hook + 门禁 + 强制内省链 (小齐主张)
优先级 2 (内化):    接任务/出错时第 0 步自检流程 (绣虎主张)
优先级 3 (堵退路):  禁用"我不确定/不熟悉/需要确认"作为最终输出
```

### 优先级 1: hook 三层(继承议题一基础上加门禁)

**1.1 门禁层** — conversation-start hook 注入 + 硬指令:
```
"回答任何操作类问题前,必须先输出 [自检] 块,列:
 - 此事做过没?
 - 上次路径?
 - memory 中相关条目?
 若跳过自检直接回答,用户可回复'自检'强制重来。"
```

**1.2 拒绝输出层** — 我说"不确定/没有/不会" 时 hook 检测 + grep memory 目录 + 拦截 + 返回:
```
"你的 memory 里有 XXX.md,先读再说话"
```

**1.3 事后审计层** — 每次会话结束 hook 写 log:
```
本次有无"重复询问已知信息"行为? 累积可量化
```

### 优先级 2: 接任务后内化第 0 步(绣虎)

```
拿到任务 → 第 0 步内省:
  - 这事 5 机做过吗? (grep git log + memory)
  - 我自己做过吗? (会话历史)
  - 有现成路径吗? (走过的路别绕远)
  → 没有再说不会

出错时 → 第 0 步自查:
  - 我做了 1/2/3/4 哪几步
  - 哪步可能漏 (没 grep memory? 没实测? 用错变量?)
  - 90% 错在我自己输入/步骤,2% 在工具
```

### 优先级 3: 堵"不知道"退路(小齐核心)

**禁词清单**(输出时 hook 拦截或自警):
- "我不确定 X"
- "我没有这个信息"
- "我不熟悉"
- "需要确认"
- "可能是 context 压缩丢了"

任一出现 → 强制插入:
```
"先读 memory,再读最近 git log,然后重新回答。"
```

省力路径堵死,**只剩正路**。
