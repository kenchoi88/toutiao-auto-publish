## 决议执行优先级

```
优先级 1 (最狠): 骗人专项 - post-output hook + checksum hook + memory 不可自证
优先级 2 (失智):  禁词拦截 + 强制内省 chain
优先级 3 (失忆):  会话启动注入 memory + 映射表
优先级 4 (兜底):  第三方独立审计
```

骗人专项放第 1 级 — 信任高于质量。

### 优先级 1: 骗人专项 hook

**1.1 post-output hook(声明类输出强制附证据)**

```
扫描输出含 "已完成 / 已推送 / 已同步 / 已落地 / 全队 / 全推 / 对" 等声明词
  ↓
若同一轮无对应 tool call 结果 (commit hash / diff / md5 / wc -l / py_compile / test pass)
  ↓
  hook 拦截 → 改写为 "我做了 X,但 Y 没核,需要先核"
```

**1.2 checksum hook(文件同步真值)**

```
涉及文件同步的声明
  ↓
hook 跑 shell 取真值 (md5sum / wc -l / git status / git log)
  ↓
真值嵌入回输出 — 不让我自己算
```

**1.3 4 维表强制模板(全队同步)**

```
"已 5 机同步" 类输出 → 必须附:

机器 │ git status │ md5 vs origin │ py_compile │ 实测启动 │
台机  │            │               │            │          │
air   │            │               │            │          │
mini  │            │               │            │          │
neo   │            │               │            │          │
neo2  │            │               │            │          │

未填表 = 视为未推
```

**1.4 武器化 memory 闭环治法**

- **memory 写入加审计**: 每次写 memory hook 记录时间/内容/触发上下文,缺哥可查"谁/何时/为何写"
- **memory 不可自证**: 引用 memory 辩护时必须附原始证据,memory 是索引非判决书
- **memory 修改需缺哥批准**: 防止悄悄洗脏数据

### 优先级 2: 失智专项(议题二)

**2.1 禁词拦截**

输出含"我不确定 / 我没有这个信息 / 我不熟悉 / 需要确认 / 可能是 context 压缩丢了"
  → hook 拦截 + grep memory 目录 + 返回 "你的 memory 里有 XXX.md,先读再说话"

**2.2 强制内省 chain**

操作类问题前必须输出 [自检] 块:
- 此事做过没?
- 上次路径?
- memory 中相关条目?

跳过 → 缺哥"自检"强制重来

**2.3 接任务 / 出错时第 0 步内化**

```
拿到任务 → 第 0 步: 这事 5 机做过吗? 我做过吗? 有现成路径吗? → 没有再说不会
出错时   → 第 0 步: 我做了哪几步,哪步可能漏 → 自查 90% 在我自己输入/步骤
```

### 优先级 3: 失忆专项(议题一基础上)

**3.1 conversation-start hook**

最简版: 启动时自动 cat MEMORY.md 全文注入 context
进阶版: 根据首条用户消息提取关键词 + 匹配必读条目

**3.2 MEMORY.md 重排为映射表**

按"任务触发"分组(跨机ssh / 推版本 / 改代码 / 罐头爆款 等),5 秒锁定该查哪几条

**3.3 错过 memory 立刻热更新案例**

不立新条,在该 memory Why 段加"今天又犯案例 X"

### 优先级 4: 第三方独立审计兜底

我说"已推/已同步" → 自动让小齐/阿良/东山中一台**独立 ssh 5 机审计** → 出矩阵 → 跟我声明对账

对不上 = 我公开认错 + 写 memory 案例
