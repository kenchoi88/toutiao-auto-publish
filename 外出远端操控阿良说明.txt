[阶段 3] 设计 + 落地 plist HTTPS_PROXY (00:42)
─────────────────────────────────────────────────────────
阿良 方案: 注 HTTPS_PROXY=http://127.0.0.1:1082 给 daemon,
     让 daemon 走 Shadowrocket HTTP 代理(CONNECT host:443),
     代理服务器自己解 controlplane.tailscale.com (走它远端 DNS),
     完全绕开 Mac 系统 fake-IP。
     原理: Go net/http 走 HTTPS_PROXY 时发 "CONNECT host:443",
           hostname 字符串原样传代理,不依赖本地解析。

阿良 改 /Library/LaunchDaemons/com.tailscale.tailscaled.plist:
     原 plist 备份: /Library/LaunchDaemons/com.tailscale.tailscaled.plist.bak.1777999342

     插入到 <dict> 内:
       <key>EnvironmentVariables</key>
       <dict>
           <key>HTTPS_PROXY</key>
           <string>http://127.0.0.1:1082</string>
           <key>HTTP_PROXY</key>
           <string>http://127.0.0.1:1082</string>
       </dict>

阿良 sudo bootout + bootstrap → daemon 带新环境变量重启 PID 61925

阿良 5 条边界 (绣虎没看到的):
    ① Shadowrocket NE 时序冲突(最大坑)
       Shadowrocket 自己也监听 network_change,会重新拉 utun9 +
       注入 tun-excluded-routes。watcher 同时触发 tailscale up,
       daemon 跑得比 NE 快时,daemon 注入 100.64/10 → utun0
       后 NE 起来又会再写一次甚至覆盖 → 我们刚修好的路由又被打回去
       对策: watcher 收到事件后必须 sleep 5-8s 再跑,
            给 Shadowrocket 自己折腾完的时间

    ② 防抖 (debounce)
       SSID 漂移 / 热点反复开关 → network_change 一秒内可能 5-10 次
       watcher 不防抖会嵌套 down/up,daemon 状态错乱
       对策: 用 PID 文件 + 时间戳, "最后一次事件 5s 内不再触发"

    ③ 断网 vs 切换的判断
       完全断网也触发 network_change,此时跑 tailscale up 浪费
       对策: 跑前先验 route -n get default 是否有可达 gateway

    ④ 路由已经对的就不跑 (关键防自我攻击)
       唤醒 / 配置变更等场景路由可能本来就好的
       对策: 先验 netstat -rn | grep '^100.64/10.*utun0',
            已经在 utun0 才跑,否则跳过

    ⑤ Kill switch (防自杀)
       万一 watcher 死循环 (每次 up 触发 network_change 又触发 watcher),
       air 立即变砖
       对策: 脚本顶部加 [ -f /tmp/ts_watcher_disable ] && exit 0
            一旦出 bug,远程 touch /tmp/ts_watcher_disable 一秒禁用

【~/.config/ts-watcher/ts-watcher.py】 (A 路 主脚本)
────────────────────────────────────────────────────
#!/usr/bin/env python3
"""Tailscale 切网自动软修 watcher (air, A 路自动化)

每 POLL 秒检查路由表,发现 100.64/10 不在 utun0 时跑
`tailscale down + up` 软修。覆盖任何切网/唤醒场景,
最坏 POLL 秒内修好。

设计要点 (跟绣虎商定的 5 条边界):
1. Shadowrocket NE 时序:发现需要修后 sleep GRACE 秒,
   等 NE 写完路由再修
2. 防抖:轮询天然串行,不存在嵌套触发
3. 断网判断:无可达默认网关跳过 (避免 daemon retry 噪音)
4. 条件触发:100.64/10 已 → utun0 时无声跳过 (无日志噪音)
5. Kill switch:/tmp/ts_watcher_disable 存在则跳过 (远程一键禁用)
"""
import logging
import os
import subprocess
import time

POLL = 30      # 轮询间隔 (秒)
GRACE = 5      # 发现需要修后等 5s 让 Shadowrocket NE 把它的路由写完再修
KILL = "/tmp/ts_watcher_disable"
TS = "/opt/homebrew/bin/tailscale"
LOG = os.path.expanduser("~/Library/Logs/ts_watcher.log")

logging.basicConfig(
    filename=LOG, level=logging.INFO,
    format="%(asctime)s %(message)s", datefmt="%Y-%m-%d %H:%M:%S",
)


def needs_fix() -> bool:
    """100.64/10 没指向 utun0 时返回 True"""
    out = subprocess.run(
        ["netstat", "-rn", "-f", "inet"],
        capture_output=True, text=True,
    ).stdout
    for line in out.splitlines():
        if line.startswith("100.64/10") and "utun0" in line:
            return False
    return True


def has_default_route() -> bool:
    """有默认路由 (interface 字段) 时返回 True;
    注意:utun9 这种 PtP 默认路由无 'gateway:' 字段,只有 'interface:',
    所以判断 interface: 而非 gateway:"""
    out = subprocess.run(
        ["route", "-n", "get", "default"],
        capture_output=True, text=True,
    ).stdout
    return "interface:" in out


def run_fix() -> None:
    if os.path.exists(KILL):
        logging.info("kill switch %s 存在,本轮跳过", KILL)
        return
    if not has_default_route():
        return  # 完全断网,无声跳过
    if not needs_fix():
        return  # 路由正常,无声跳过 (避免日志噪音)

    logging.info("检测到 100.64/10 不在 utun0,等 %ds Shadowrocket NE 安顿", GRACE)
    time.sleep(GRACE)
    if not needs_fix():
        logging.info("等待期间路由自己恢复,跳过")
        return

    logging.info("开始软修 (tailscale down + up)")
    try:
        subprocess.run([TS, "down"], timeout=10)
        time.sleep(2)
        up = subprocess.Popen([TS, "up"])
        time.sleep(5)
        if up.poll() is None:
            up.terminate()
        time.sleep(2)
    except Exception as e:
        logging.warning("软修异常: %s", e)
        return

    if not needs_fix():
        logging.info("✓ 软修成功,100.64/10 → utun0")
    else:
        logging.warning("✗ 软修后路由仍未恢复,需人工")


def main() -> None:
    logging.info("=== ts-watcher 启动 (PID=%d, 轮询 %ds) ===", os.getpid(), POLL)
    while True:
        try:
            run_fix()
        except Exception as e:
            logging.warning("主循环异常: %s", e)
        time.sleep(POLL)


if __name__ == "__main__":
    main()

    /opt/homebrew/bin/tailscale down
    sleep 2
    echo "[tsfix] tailscale up ..."
    /opt/homebrew/bin/tailscale up &
    local UPPID=$!
    sleep 5
    kill $UPPID 2>/dev/null
