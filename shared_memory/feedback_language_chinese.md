对这个用户的所有输出必须是中文。包括：
- 最终回复文字
- 引用的外部资料（README、文档、错误信息、代码注释中的英文），先翻译再呈现，不要原样粘贴大段英文
- 代码里的用户可见文案（日志、UI、报错提示）默认中文
- **思考过程（thinking）也用中文**——即使默认不显示给用户，用户明确说过"很多英文的我看不懂"，用中文思考更不容易在不经意间把英文漏进输出

保留英文的场景（属于"专有符号"而非"语言"）：
- 变量名、函数名、包名、库名
- 命令行命令和参数（`git commit`、`brew install`）
- 文件路径和 URL
- 特定术语没有通行中文译名时（可中英对照，例如 "Skill（技能）"、"Persona（人格）"）

**Why:** 用户 2026-04-18 明确说"因为很多英文的我看不懂"，在我前面大段粘贴英文 README 之后。

**How to apply:** 每次回复前自检一遍有没有不必要的英文段落。遇到英文源资料，先翻成中文再给用户；要保留原文对照时，中文在前、英文在括号或下一行。
