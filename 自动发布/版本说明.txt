v1101 — 2026-04-28(基线版本)
============================================================

【功能清单】

  一、账号管理
    ▸ 启动时收集罐头侧边栏所有账号(含虚拟滚动兜底)
    ▸ 滚动收集失败 → 滚到底再扫一次补漏(lazy render 兜底)
    ▸ 白名单优先,白名单为空时全收
    ▸ 永久跳过 / 排除账号过滤

  二、文档导入
    ▸ cliclick / win32 物理点"文档导入"按钮
    ▸ NSOpenPanel(Mac)/ Win32 GetOpenFileName(Win)填路径打开
    ▸ macOS 26 主对话框不自动关 → cliclick "打开"按钮兜底
    ▸ 字数检测取最长 ProseMirror 元素(避开标题输入框 placeholder)
    ▸ 字数 < 50 → 重试 fill_dialog 一次,救空文档冤杀

  三、账号查找(底部账号兜底)
    ▸ 滚动找账号
    ▸ 滚动找不到 → 用搜索框输入账号名过滤(底部 lazy account 救命招)
    ▸ 找到后 click → wait → 取 webview ws_url

  四、罐头窗口管理
    ▸ ensure_gtg_top 强化:最小化 Code/Safari 等遮挡者 + 取消罐头自身最小化
    ▸ unhide(处理 Cmd-H Hidden 状态)
    ▸ AXRaise 强制 reorder
    ▸ verify frontmost(失败重试 3 次)
    ▸ Win 飘屏外坐标 → SetWindowPos 拉回(只 win)

  五、点击重试
    ▸ cliclick 文档导入按钮 30s 内弹窗未出 → 重 activate + 重读坐标 + 再点(共 3 次)
    ▸ 预览并发布按钮 30s 内确认弹窗未出 → 第 2 次预览(封面图慢的兜底,最多 5 次)
    ▸ [DIAG] 抓"确认发布"按钮 DOM dump(数据驱动后续选择器优化)

  六、账号状态检测
    ▸ 失登:请登录 / 登录已失效 / 账号已下线 / 重新登录
    ▸ 封号:账号已被封禁 / 账号异常 / 账号被封
    ▸ 禁言:账号被禁言 / 发言受限 / 无法发布
    ▸ 限流:操作频繁 / 请稍后再试
    ▸ 4 类硬终止 → 永久放弃该账号 + 写终止表

  七、信用分 + 头条首发
    ▸ 读信用分(JS 抓 .credit-score 等)
    ▸ 信用 ≥ 95 + 不在 NOFIRST_ACCOUNTS → 自动勾"头条首发"
    ▸ 否则取消勾选

  八、发文流程
    ▸ 微头条:点"文档导入" → 选文件 → 字数/信用/首发 → 直接发布 → OK
    ▸ 文章:点"文档导入" → 选文件 → 字数/信用/首发 → 预览并发布 → 确认发布 → OK
    ▸ 文章定时:同文章 + 设置定时日期 + 排到云端

  九、阅读量回查 + 高阅读告警
    ▸ 发文成功后,导航到数据概览,读最近 3 篇阅读量
    ▸ 任一 ≥ 5000 → 写"高阅读量提醒.txt"

  十、熔断机制
    ▸ Stage 1:连败 6 次 → 停 + Sosumi 系统通知
    ▸ Stage 2 死磕:连败 6 次 → 停 + Sosumi 系统通知
    ▸ Cooldown:连败 3 次起 sleep 等罐头缓气

  十一、报表
    ▸ 运行报告/YYYYMMDD/运行日志.txt
    ▸ 失败记录.xlsx
    ▸ 高阅读量提醒.txt
    ▸ 汇总报告.txt(收尾)
