--- [00:30~01:15] [air] [Tailscale 接入] [Shadowrocket TUN fake-IP 劫持 daemon 控制面注册 + 路由 + SSH 互信三连失] ---
现象: 缺哥晚上准备让 air 重接 tailnet,5 个 tailscale 进程残留并存(2 tailscaled + sudo 卡 stdin + 多个 up 卡等)。
      绣虎远程清理后 daemon 干净起步,但 tailscale up 仍不吐认证 URL,30 秒 timeout。
      接入后 air → mini/neo/neo2 SSH 全 Operation timed out + Permission denied。
影响: air 完全无法加入 tailnet,跨机 SSH/scp/dispatch.py 推送链路对 air 失效约 5 小时(出移动热点回家后断到本会话开始)。
上下文: air 出门接移动热点 → 回家接 AX3 局域网 → Shadowrocket TUN 配置切换 → tailscaled 试图重连 controlplane 拿不到。
推测根因: 三层叠加问题——(a) Shadowrocket fake-DNS 劫 controlplane.tailscale.com → 198.18.x;
          (b) 接入后 100.64.0.0/10 路由被 utun9 抢,跨机 TCP 走 192.168.3.1 局域网网关丢包;
          (c) air reauth 拿新 NodeKey 但 SSH host key 没自动同步,authorized_keys 对端不认。
谁记的: 阿良(air) + 绣虎(台机)
状态: 已修

★ 修后补:

修法(三件):
  1. plist HTTPS_PROXY 注入(解 daemon 控制面被 fake-DNS 劫):
     /Library/LaunchDaemons/com.tailscale.tailscaled.plist 加 EnvironmentVariables:
       HTTPS_PROXY=http://127.0.0.1:1082
       HTTP_PROXY=http://127.0.0.1:1082
     bootout + bootstrap 重启 daemon,4 秒吐认证 URL。
     原理: Go net/http 走 HTTPS_PROXY 时发 CONNECT controlplane.tailscale.com:443 给代理,
           hostname 原样传 Shadowrocket,代理自己解 DNS 不走 Mac fake-IP。
     备份: /Library/LaunchDaemons/com.tailscale.tailscaled.plist.bak.1777999342

  2. tailscale down/up 软修路由(无需 reboot,跟 04-28 mini 必须 reboot 的实战不同):
     tailscale down + tailscale up,daemon 重新注入 100.64.0.0/10 → utun0 路由表,
     从 "default → 192.168.3.1 / en0" 改成 "100.64/10 → utun0",跨机 TCP 立通。

  3. SSH 互信修复(绣虎从台机一次性灌,阿良不动):
     台机 ssh 进 mini/neo/neo2 把 air 当前 ed25519 pubkey append 进 authorized_keys。
     air 新 pubkey: ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIEr/8sHfdQVwVHzlZ4C6fu16AHXIHG/LhGxuK5uk5J9J kenair@Air

  4. 顺手补漏 mini default.db docid=2 缺 100.64.0.0/10 的 skip-proxy:
     UPDATE config SET value 追加 100.64.0.0/10 到第 4 位(跟 air 顺序一致)。
     air 还多加了 docid=667 IP-CIDR,100.64.0.0/10,DIRECT,no-resolve 作 Rule 段第三层兜底。
     摸出 Shadowrocket Rule 评估顺序按 c6created DESC 不是 docid。
