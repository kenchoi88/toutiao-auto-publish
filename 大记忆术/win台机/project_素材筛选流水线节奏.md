罐头爆款素材每日批次走三轮筛选,然后供下一日发文使用。

**三轮筛选量级(2026-04-26 实测):**
1. 原始批次 ~2000 条
2. v0 分类器(我的删法_v0.py)自动删 ~200 → 剩 ~1800
3. Claude 二轮筛(对剩余按累积黑名单再过)删 ~200(本次只删了 16+,但用户原话第一次挑出 200 多) → 剩 ~1600
4. 用户人工三轮筛删 ~200(本次工作流) → 剩 **~1500 条**

**1500 条 ≈ 一天发文量**(覆盖所有账号微头条 5/号 + 文章 3/号 + 定时 3/号)。

**节奏(2026-04-26 案例):**
- 4-26 下载 → 三轮筛 → 4-28 发文用
- 4-27 已经有上一批洗好的稿,4-27 当天直接发
- 4-28 当天还要**再下载一次**罐头爆款补充,因为 1500 条不够 28 号当天总量

**Why:** 罐头爆款单次下载上限就是 ~2000 条,过完三轮筛只剩 ~1500,不够某日大量发文,所以需要"前一天素材池 + 当日早上再下一批"两批拼起来。

**How to apply:**
- 收到新批次时,默认假设是给"后天"用的(D+2),不是当天发
- 看到批次量 < 1500 别紧张,正常水位;< 1000 才需要警告用户量不够
- 每天早上提醒/检查是否需要再下载一批补充
- 三个文档([素材筛选_剔除记录.md](素材筛选_剔除记录.md) / [素材筛选_保留候选.md](素材筛选_保留候选.md) / 工作版 xlsx)在归档时按"用于 YYYY-MM-DD 发文"打标签,方便溯源
