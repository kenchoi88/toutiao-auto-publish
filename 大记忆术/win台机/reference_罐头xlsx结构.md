## 关键坑

### 坑 1: 「标题」列不是标题,是全文

如果代码里 `header.index('标题')` 取出来的值 100+ 字,那是正文不是标题。
**没有「内容」列** — 罐头把全文塞在「标题」列里。
之前 `我的删法_v0.py` 的 `classify_xlsx` 误以为有「内容」列要拼接,实际只读「标题」列即可,里面就是全文。

### 坑 2: 用罐头自带「领域」字段做板块筛选,不要 keyword 推断

2026-05-06 我犯过的错:写 `classify_sector(text)` 用 keyword 推断板块(国际/军事/教育/职场职业),实测发现严重误归类:
- "浏阳烟花厂"(三农领域,我推到职场职业,因为含"加班"词) ❌
- "走亲戚见闻"(美食领域,我推到职场职业,因为含"职场"词) ❌

**正确做法:** 直接读 col 4「领域」,精度 100%。

```python
domain_idx = header.index('领域')
DOMAIN_TO_SECTOR = {
    '国际': '国际',
    '军事': '军事',
    '教育': '教育',
    '职业职场': '职场职业',  # 注意罐头叫"职业职场",缺哥习惯说"职场职业",同义
    # ...
}
for row in rows:
    domain = row[domain_idx].value
    sector = DOMAIN_TO_SECTOR.get(domain)
```

### 坑 3: 「职业职场」 vs 「职场职业」 命名不一致

- 罐头自己用「**职业职场**」(职业在前)
- 缺哥习惯说「**职场职业**」(职场在前)
- 19 板块清单见 `罐头爆款下载.py` 顶部 `TARGET_DOMAINS`

代码里映射时两者都接受。
