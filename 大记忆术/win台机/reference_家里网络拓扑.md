## 当前拓扑(2026-05-02)

```
电信光猫 ──┐
          └─ 华为电信 AX3 ── 出口走电信 ★平时用
                
移动光猫 ──┐
          └─ 华为移动 Q6  ── 出口走移动 (备用,平时不跑发文)

设备(台机 + 4 Mac):全员接 AX3 走电信
跨机通信:走 Tailscale 100.x,与局域网解耦
```

## 关键事实

- **平时用电信(AX3),移动 Q6 仅备用**(2026-05-02 缺哥拍)
- 两台华为路由器都在用,分别接两条 ISP
- 移动 Q6 只在电信故障/特殊场景才用,正常发文不走
- 出口风控差异客观存在 — 切到移动会触发账号风控,见 [出口运营商切换风险](project_出口运营商切换风险.md)

## 命名规范(避免再混淆)

- **华为电信 AX3**:接电信光猫,平时用
- **华为移动 Q6**:接移动光猫,备用
- 不再用旧称"小旋风/全 Q6/Q6 主路由"等

## 历史回溯

- 2026-04-27 小旋风(华硕)落地版,全员走电信 → 04-29 退货决定
- 2026-04-30 小旋风退货,Q6 临时做主路由,4 Mac 走 Q6 移动
- 2026-05-01 华为 AX3 到货接电信光猫
- 2026-05-02 双华为并存确认,平时用电信 AX3,Q6 移动备用

## How to apply

- 默认所有发文设备都走 **华为电信 AX3**
- 临时切到 **华为移动 Q6** 是异常场景(电信故障 / 单独测试),要意识到会触发出口运营商切换风控
- 跨机文件 / SSH / 状态汇报继续走 Tailscale 100.x,不依赖局域网网段
- 看到设备发文异常,先确认它接的是不是电信 AX3
