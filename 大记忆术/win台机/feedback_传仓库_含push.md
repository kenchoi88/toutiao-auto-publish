**适用范围:5 台机器(Win 台机 / air / neo / neo2 / mini)全部一致,无例外。**
**生效时间:2026-05-01 起立即生效。**

## 核心规则

1. **每改必推** — 任何文档/手册/规则/计划/笔记的修改或新建,**改完立刻** push 到云端 origin,
   不批量、不积压、不"等想完整再推"。
2. **每创必推** — 新写的文件同样立刻走完 add → commit → push 全链路。
3. **不存散文件** — 桌面、Documents、Downloads、临时目录等"非 git 工作树"位置,**零副本**。
   git 工作树本身受 git 管理,是允许的镜像;散文件是"git 之外的副本"。
4. **唯一权威 = 云端 origin** — 同一份内容多机各自一份的散副本无效;以 origin 最新版为准。

## 触发词(任一出现即走完整链路)

"传仓库 / 放仓库 / 上仓库 / 推仓库 / 推上去 / 上云 / 落仓"
"整理成文档 / 沉淀下来 / 写个手册 / 落规范 / 立规则"
"以后给 X 看 / 给全队读 / 给外出的人读"
以及——**任何对 git 工作树内文件的修改/新建,无需触发词,默认就要 push**。

## 完整链路

```
1. Write/Edit 到 ~/code/<repo>/ git 工作树(不是桌面、不是 Documents)
2. git add <file>
3. git commit -m "<一句话:为什么改 + 给谁用>"
4. git push 到 origin
5. 简短回报:"已落工作树 + 已推 origin · <commit hash 前 7 位>"
```

## 写文档的目录优先级

- 跨机通用规则/手册 → `~/code/头条自动发布/`(仓库根)
- 共享知识库       → `~/code/头条自动发布/shared_memory/`
- 单机专属脚本/配置 → `~/code/头条自动发布/<机器名>/`(win台机/air/neo/neo2/mini)

## 5 机各自适用方式

- **Win 台机(我=绣虎)**: 改完直接走链路;push 是默认动作
- **air(阿良) / neo(小齐) / neo2(左右) / mini(东山)**: 各机 Claude 同样规则,改完即推;
  外出/远程时优先 `git pull` 取最新,改完 `git push` 同步全队
- 任意机的 Claude 看到本规则,直接遵守,不必征询缺哥

## Why

- 仓库是 5 机 + 云端(暖树/景清)协作的唯一同步枢纽
- 不即时推 → 4 Mac 看不到 → 漂移失控,排查时不知信哪份
- 散文件 → 各机各一份,几天就乱;加上"桌面是真仓库是镜像"规则会冲突
- 历史教训:多次出现 air/neo/Win 各存一份手册,版本不一致

## 例外清单(已既成事实,不强行迁)

- **GTG 三大件运行真版** 在 `~/Desktop/台机专用自动发布/<件>/`(参见 feedback_桌面才是真版.md);
  这是**代码运行位**而非文档,工作流"桌面 → 跑验证 → cp 仓库 → push"已稳定
- **MCN 数据 xlsx** 在 `~/Desktop/MCN数据下载/`(数据文件,日产日清,不入仓库)
- **罐头爆款下载** 在 `~/Desktop/罐头爆款下载/`(同上,日产数据)
- 缺哥临时双击运行的 `.bat` 快捷启动可放桌面(快捷方式,不是真版)

## 异常处理

- push 失败/commit 冲突 → 告知缺哥实情,不隐瞒、不静默修复
- 大量改动一次性推不动 → 拆分多个小 commit,每个都 push,不要打包压成"等下次一起"
