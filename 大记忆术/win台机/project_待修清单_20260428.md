## 2. neo SSH 不通(2026-04-28 10:30 实测)

**症状:** Tailscale ping neo 通(direct 100.68.57.96 / 192.168.50.74:41641),TCP 22 端口能连(exit 0),但 OpenSSH banner 不出 → connect timeout / banner exchange 超时。其它 3 mac(mini / air / neo2)SSH 全正常。

**Tailscale admin 状态:** mini / air / neo2 有 SSH tag(Tailscale SSH 开),neo 没开。我们走的是普通 OpenSSH,这个 tag 没影响 — 真问题是 neo 自身 sshd 挂了或 Remote Login 关了。

**修法候选:**
- (a) neo 上 GUI 开「系统设置 → 共享 → 远程登录」(最稳)
- (b) 在 admin console 给 neo 打开 Tailscale SSH tag(绕开 OpenSSH)
- (c) ssh -v 抓 banner 阶段错误码,看是 EOF / RST / 静默丢包

**Why:** neo 是小齐+小师弟所在,publish 脚本(带"空文档重试"参考实现)在 neo 上,跨机改 mini gtg 时需要参考 neo 代码。SSH 不通 = 断了主要参考途径。

**How to apply:** 缺哥本机直接 ssh neo 走原来的命令,不通就先用 (a) 或 (b);(c) 是查根因。

## 4. B 类:文档导入弹窗未出现(Mac 散点 + Win 飘屏外合并)

### 4a. Win 台机:罐头窗口飘屏外

**症状:** webview 原点变成大负数(实测 win 定时 04-28: -25384, -25508),cliclick 算出来的屏幕坐标也是负的(-24348, -25391),物理鼠标点到虚空,弹窗永远不出。

**实证(2026-04-28):**
- win 定时 01:53 - 09:08 持续 7 小时 85 次连续失败
- 时间线锁定:01:51:35 取消首发 cliclick 后立刻读到的下一个坐标就飘了
- 触发猜测:取消首发 cliclick 副作用 / 罐头自身定时事件 / Win 11 多显示器配置变化
- 跨机搜索:其它 4 mac 4 天 log 0 次负坐标,目前 win 专属

**修法候选:**
- (a) 脚本检测 webview 原点为负 → 立刻 Win32 `SetWindowPos` / osascript `set position of window 1 to {100, 100}` 拉回再重读坐标
- (b) `ensure_gtg_top` 加坐标合法性 check(任何坐标 < 0 视为窗口异常,触发拉回)
- (c) 加熔断:连续 N 次坐标异常自动停 + 系统通知

**Why:** 缺哥昨夜没盯,7 小时 85 次失败 — 跟 Stage 2 死磕缺熔断同款问题(不停尝试). Mac 4 天没观测到这种集中飘屏,但 Win 11 + Electron 罐头有这风险,Mac Electron 同样可能(macOS Stage Manager / 多屏切换)。

**How to apply:** Win 用 `pywin32 SetWindowPos`,Mac 用 osascript。统一加在算坐标的函数前。

### 4b. Mac cliclick 弹窗未出现(主战场)

**症状:** cliclick 点击文档导入,坐标正常(无负数),罐头活着,但 30s 内"选择文档"弹窗就是不出 → 报"文档导入弹窗未出现"。脚本不重试 cliclick,直接报失败。

**4 天数据(去除迁网事件后):**
- 4 天 Mac 全队约 77 次(全 4 天 175 - 04-27 neo2 12-14 时迁网 98 = 77)
- 散布所有时段,平均每天每机 ~6-9 次

**根因猜测:**
- cliclick 偶发被罐头 React 拒识(isTrusted 检查 / focus 不对 / 罐头处理上一动作未完)
- 罐头 toolbar SPA 渲染晚于 ensure_gtg_top
- 罐头某 sub-page 上没"文档导入"按钮,cliclick 打到空区

**修法:**
- 30s 内"选择文档"未出现 → cliclick 再点 1-2 次(每次重读按钮坐标 + ensure_gtg_top 后再点)而不是直接报失败
- 配合 #3 加 frontmost verify,cliclick 前先确认罐头真前台

**Why:** Mac 散点偶发 6-9 次/天/机,失败篇直接回池等下个账号 — 累积下来一天少发几篇,重试能救大半。

**How to apply:** mini gtg + air gtg_batch + neo2 gtg_timer + 三大件 gtg_batch 全要改;改 publish_article_timer 里 `if not v: return False, "文档导入弹窗未出现"` 之前加 retry 包装。

**症状:** Win 台机 ping neo2(100.96.153.17)100% 丢,但 tailscale ping LAN direct(192.168.50.48:41641, 4ms)+ TCP/22 通,SSH 可用。其余 3 mac(air / neo / mini)ICMP 全通。下午曾全通晚上掉,推测 neo2 macOS 层(pf 或 firewall stealth mode)在掐 ICMP echo。

**缺哥定调(2026-04-28 晚):** SSH 能过去就够用,不急修;等 neo2 上左右 Claude 上线,让左右自查 + 处理。绣虎别越界跨机改系统配置。

**How to apply:** 左右上线后,把"自查 stealth mode + pf 规则,放行 ICMP echo"作为 onboarding 第一项交给左右自己跑。绣虎不主动 SSH 进 neo2 改东西。
