**事件:** 2026-05-11 平台 5/8 创作指引明确 "AI 生成内容未标注 = 失实表述扣 10 分 / 篇", 缺哥拍 5 机微头条加自动勾选「作品声明:取材网络」复选框, V1102.7 一次过推完。

**Why:** 缺哥发的微头条多用 AI 辅助生成, 没勾「取材网络」就被扣分。一次过急推, 不分批 NEO2 试 — 漏一篇扣 10 分比 quota 漏 1 篇更糟。

**实现核心 (publish_article 流程内插入):**

调用层位置: 头条首发处理后 / 点击发布前
```python
head_first_handle(ws, ...)
if not ensure_zuopin_shengming_checked(ws):
    wsc.close()
    return False, "作品声明勾选失败"  # 单篇硬保护跳过
# 点击发布
```

函数核心:
1. JS 找 `textContent` 含「取材网络」+ `length<20` (防误命中长文本) 的 label/span/div
2. 找父级或兄弟 `input[type=checkbox]`
3. `cb.click() + cb.dispatchEvent(new Event('change', {bubbles: true}))` 兜底
4. `time.sleep(0.4)` 后 query 一次 `cb.checked` 回读校验 (React 受控组件需异步)
5. 失败 → return False, "作品声明勾选失败" → 单篇跳过

**关键设计 (跟 v1101.3 头条首发硬保护同款套路):**
- 失败**不进 HARD_TERMINATE_REASONS** — 该号只是这一篇不发, 下次重试可能 DOM 加载好就过, 不永久放弃账号
- 失败**算硬保护跳过**, 该篇报废 — 防漏勾被扣 10 分
- 平台业务铁律: 不发出去 = 0 分, 发出去未勾 = -10 分 → 失败该弃篇
