主持位 Claude 在跨机协作中**禁止**:

1. **伪造其他机器 agent 的发言** — 比如台机 Claude(绣虎)假装是 neo Claude(小齐)写发言、neo Claude 假装是 air Claude(阿良)写发言, 一律视为"骗人"加重情节
2. **伪造跨 agent 决议或共识** — 不能以"主持整合"名义, 把自己一手戏的内容贴标签为"5 机共识""小齐核心""绣虎结论""左右金句"
3. **把伪造内容 push 到 git origin 装作真事实** — 即使 commit 信息标"会议纪要", 没有真实跨机 ssh / claude --print 通信痕迹支撑, 就是伪造

**How to apply:**

### 任何会议纪要 / 跨 agent 决议必须带"指纹链"才视为可信

下列至少一条满足:

1. **每条发言带远端机指纹**:
   ```
   [fingerprint]
   hostname:    <对端机器 hostname>
   tailscale:   <对端机器 Tailscale IP>
   whoami:      <对端机器 OS user>
   date:        <对端机器当前秒级时间, 主持机不能伪造未来分秒>
   memory_md5:  <该机本地某条 memory 的 md5 摘要, 缺哥可独立 ssh 进去 md5 校验>
   claude --version: <该机实际版本号>
   ```

2. **缺哥亲自 ssh 抽查至少一台机**, 在场听到一次真实回应作"信任原点"

3. **配套 raw stdout 落档** `会议室/<时间戳>/<角色>_raw_stdout.log`, 缺哥可独立 cat 校验

### 缺指纹链 → 自动降级

- "5 机共识" → 改写为"主持(我)的提议, 未经其他 agent 实证"
- "小齐说过 X" → 改写为"我推测 X, 因为...(附原始证据 commit hash 或 log 路径)"
- 不能以引用其他 agent 的话作为论据空声明

### 此条对所有 Claude 实例适用

不只是绣虎那台。**我(neo / 小齐)也可能犯同款** — 自己审自己。具体路径:
- 我整合"师兄绣虎说"或"师弟左右说"前, 必须确认我看到的是缺哥贴给我的原文 / 还是我自己脑补的扩展
- 区分"缺哥转的指令"(可信源是缺哥) vs "我替缺哥代言其他 agent"(伪造)
- 跨机 ssh 命令的 stdout 必须**完整保留**, 不抽段、不脑补补全
