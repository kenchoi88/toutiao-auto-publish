```
自动发布/自动发布V1102.X/{件}V1102.X/{mac,win}/
```

|---|---|---|
| 微头条自动发布 | V1102.2(V1102.3 不动) | `自动发布/自动发布V1102.2/微头条自动发布V1102.2/{mac,win}/gtg_batch.py` |

## ⚠️ 不算标答的目录(历史镜像/老版本)

`仓库根/`下这些是**镜像/老版本归档**,跟"云端唯一标答"对应不一致很正常,**不要混入审查矩阵**:
- `win台机/GTG_青春小馆{自动发布微头条,自动发布文章,定时自动发布}/` — 绣虎本机镜像(可能 stale)
- `MAC数据/{air,mini,neo,neo2}/{自动微头条,自动文章,自动文章定时}/` — 历史快照(可能空 .gitkeep)
- `自动发布/自动发布老版本/` — 历史版本归档
- `自动发布/{微头条,文章,文章定时}自动发布101.X版/` — 旧版本号归档

## 各机桌面真版位置(配套, 谁都该跟标答字节级一致)

详见 [reference_4mac真版位置.md](reference_4mac真版位置.md) 各机桌面路径。
统一约定:**桌面真版 = 标答(去 CRLF)** — 一致是绿灯, 不一致就是真有偏差。

## CRLF/LF 提醒

Win 台机桌面 = CRLF, 仓库归档 = LF。同一份代码 md5 永远不同, **必须 `tr -d '\r' < 桌面 | md5` 才能跟仓库归档比对**。
size 差 == 行数 → 纯 CRLF 差异(字节级一致); size 差 ≠ 行数 → 真有内容偏差。

## Why 这条记忆存在

2026-05-07 左右审三大件时一开始把仓库 `win台机/` 镜像跟云端 V1102.2 标答混入同一矩阵, 误指控绣虎"漏 push"。缺哥纠正:仓库 win台机/ + MAC数据/ 这些是历史副本,不算 V1102.X 体系。**审查只对 `自动发布V1102.X/{件}V1102.X/{mac,win}/` 这份云端唯一标答**。

## How to apply

- 跨机审"是否同步" / "是否落地某版本":拿这条 reference 锁定标答位置, 再 5 机桌面 vs 标答比对。
- 任何"标答"声明:必须在版本说明里明确写云端归档目录路径; 没明确的要先问/补。
- 升新版 V1102.4 / V1103 时,标答位置约定继续 = `自动发布/自动发布VX.X/{件}VX.X/{mac,win}/`。
