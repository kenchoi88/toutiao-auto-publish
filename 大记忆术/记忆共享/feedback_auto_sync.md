## 规则
完成一个有意义的代码改动（bug修复、新功能、脚本调整）后，**主动**执行：
1. `git add` 相关文件
2. `git commit` 带清晰的消息
3. `git push` 到远端

不用等缺哥说"同步"或"提交"。

## 例外情况
- 半成品、调试中的改动 → 先不提交，等稳定
- 记忆文件（`~/.claude/` 下）→ **不进仓库**，各 agent 记忆各自独立
- 数据文件（xlsx、原始快照等）→ 已在 .gitignore 排除

## 触发时机
- 修完一个 bug / 补齐一个功能 → 提交
- 任务告一段落（用户说"搞定"/"完成"/类似）→ 提交
- 明显的小里程碑 → 提交

不要累积太多改动再一次提交；也不要每行改动都提交——以"一个完整思路的修改"为单位。

**Why:** 2026-04-19 缺哥问"以后每次都需要我说同步才会同步吗"，明确希望主动
**How to apply:** 改完代码评估是否到了"完整思路"点，是就 push
