## 2026-04-22 22:00 小齐改版要点

原来 `定时发布.xlsx` 逐行写"账号 + 发布时间 + 文档"的模式全砍掉。现在：

### 定时发布.xlsx 只剩 B1 一格要填
- A1: "发布日期"（标题）
- **B1: 要发的日期（例 2026-04-23）** ← 每次只改这一格
- 备份 `.bak_oldschema_20260422_220535` 留在同目录

### 账号清单搬到 账号配置.xlsx（与批量发同一份）
定时版和批量版现在共用同一份 `账号配置.xlsx`，sheet 结构完全一致。

### 自动排程规则（`_expand_tasks`）
脚本从 B1 日期 + 账号配置读到的账号列表 + 每账号发文数，自动展开成带时间的任务：

| 窗口 | 起始时间 | 谁发 |
|------|---------|------|
| 早 | 08:01 | 发文数 ≥ 1 的账号各 1 篇 |
| 中 | 12:01 | 发文数 ≥ 2 的账号各 1 篇 |
| 晚 | 19:01 | 发文数 = 3 的账号各 1 篇 |

- 账号间隔 `GAP_MIN=1` 分钟
- 按 Excel 顺序
- 超出窗口自然延后不截断

### 账号来源优先级（main 主流程）
1. **账号配置.xlsx「白名单」非空** → 只发白名单列出的账号（A=账号名，B=发文份数 1/2/3）
2. **白名单为空** → 从罐头侧边栏自上往下动态读所有账号，默认每个 q=3（早中晚各一篇）
3. 两种来源都会被「永久跳过」sheet 过滤

## "永不发文"账号 = 尾号弃子（缺哥 2026-04-22 告知）

罐头侧边栏最底部的账号因懒加载视口坐标问题**必定定位失败**（老问题，账号越多越明显）。

**解法**：在罐头里建一个叫"永不发文"的废账号，**固定放侧边栏最底**当弃子，吸收这个坐标 bug，保护真账号。

这取代了之前 memory 里记录的 `accounts[:-1]` 去尾策略——现在不用去尾，让脚本正常发到最后一个必败账号，反正是"永不发文"这个废号。

## 定时发 → 批量发 两段式接力（通过「待补漏」sheet）

缺哥设计的完整工作流：

### 阶段 1：定时发（gtg_timer.py）
- 67 账号 × 3 篇 = 201 个任务分早中晚三窗跑
- 跑完 `_finalize_config` 收尾：
  - **清空「白名单」**
  - 漏发账号写进**「待补漏」sheet**，字段：账号名 / 漏发数 / 失败文稿名 / 生成时间
  - 漏发数 = quota − 本次提交成功数

### 阶段 2：批量发补漏（gtg_batch.py）
- `gtg_batch.py` 启动时先读「待补漏」sheet
- 非空 → **进补漏模式**：直接把「待补漏」的账号作为发文对象，**漏发数当配额**，只补漏不多发
- 素材比待补漏记录新时，作废待补漏跑新一轮
- 相关代码位置：`gtg_batch.py` line 1776-1813

### 术语澄清（防止 agent 误解）
缺哥口头常说"漏发进白名单"——这是**简称**，实际代码里走的是**「待补漏」sheet**。两端接口对齐在「待补漏」上，不在「白名单」上。白名单在定时发完跑完会被清空。
