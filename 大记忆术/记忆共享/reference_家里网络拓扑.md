**物理布线(2026-04-27 小旋风到家后,缺哥定):**

```
仓库(房间名):光猫(电信)
   └─ 一根网线 → 房间
房间:
  小旋风(华硕)接光猫线 ── 主路由,所有发文设备走它
   ├─ WAN 口(蓝)── 接光猫拉来的网线
   ├─ LAN 口(黄)→ 台机
   ├─ LAN 口(黄)→ mini
   ├─ LAN 口(黄)→ air
   ├─ LAN 口(黄)→ neo
   ├─ LAN 口(黄)→ neo2
   └─ 2.5G 口   ── ⚠️ 留给 NAS,暂时不接

华为 Q6(移动):备用,平时不跑发文
```

**接口规则:**
- 光猫线 → 小旋风 **WAN 口**(蓝色,单独一个)
- 5 台机 → **普通黄色 LAN 口**(LAN1/2/3/4)
- 2.5G 口 → 留给后续 NAS,**当前任何机器都不接**

**结论:**
- **全员走电信**(经小旋风出口)
- 4 Mac 都从「Q6 移动」迁到「小旋风电信」
- Q6 留着做兜底,正常发文不走它
- 台机的 WiFi 之前接 Q6,迁完后是断 / 留备用 / 改连小旋风,待定

**Why:** 之前的"双 ISP 并存"是过渡形态(小旋风没到时);小旋风到了之后缺哥要全员统一走电信,运营商一致避免账号风控差异,且电信对字节后端更友好。NAS 后面到位独占 2.5G 走全速。

**How to apply:**
- 看到任意设备发文失败,**先确认它是不是已迁到小旋风**(新 IP 段),还在 Q6(192.168.10.x)的属于过渡态
- 跨机访问(SSH/SMB)走小旋风新网段,旧 known_hosts 里的 192.168.10.x 大多会失效,要重抓 IP
- mini 的 webview 失败可能就是出口换电信后会自愈,迁完先观察
- Q6 备用线路保留,真出事可以临时切回去发文
- **2.5G 口预留给 NAS**,任何设备都别占——别为了"图快"把台机插上去
