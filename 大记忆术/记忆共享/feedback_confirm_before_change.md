网络 / 系统配置 / 代理 / IPv6 / DNS / hosts / 注册表 / v2rayN 参数 等本机环境类改动，一律**先查清楚再提方案，用户确认后才能动手**。不要"顺手就改了"。

**Why:** 本机存在多个互相牵制的配置（IPv6 开关、系统代理残留、v2rayN 模式、VS Code/Claude 出站路径……）。改动一处常破坏另一处 —— 历史已发生：阿良关 IPv6 修好了国内视频，但同时把 VS Code 连 Claude 的路径也弄坏了。擅自动手 = 把用户拖进二次抢修。

**How to apply:**
- 只读排查：进程、端口、网卡绑定、路由、DNS、hosts、注册表代理项、v2rayN 配置文件、Claude Code settings.json、事件日志等。
- 汇总现状 + 候选方案（并标清每个方案的**副作用**和**可能弄坏的东西**）。
- 等用户明确说"动手" / "执行" / "改" 再做。模糊的"嗯""好"不算授权。
- 改完第一时间回报，不要静默。
